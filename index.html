AnimePotik-MPA

Мій четвертий опублікований проєкт. Він представляє собою сайт для перегляду аніме, підключений до Jikan API.

Основні відмінності AnimePotik-MPA та AnimePotik-SPA:

  • Архітектура: AnimePotik-MPA на архітектурі MPA (детальніше про різницю архітектур читати на AnimeHub-SPA).
  • Дизайн: AnimePotik-MPA має покращений дизайн, в порівнянні з AnimePotik-SPA.
  • Інше: Крім невеличких технічних покращень та вищесказаного, в іншому AnimePotik-MPA такий же як і AnimePotik-SPA.

AnimeHub-MPA

Мій третій опублікований проєкт. Він представляє собою сайт для перегляду аніме, підключений до Jikan API.

Основні відмінності AnimeHub-MPA над AnimePotik-SPA:

  • React та додаткові бібліотеки: AnimeHub-MPA написаний з використанням фреймворка React для сучасноі архітектури та бібліотек Font Awesome для іконок і Marked для транслювання мови розмітки Markdown в html. AnimePotik-SPA написаний на Vanilla JavaScript, без використання фреймворків та бібліотек.
  • Інформація: AnimeHub-MPA крім стандартної інформації про аніме також підтримує детальну інформацію про персонажів аніме та акторів/актрис які озвучували даного персонажа. AnimePotik-SPA підтримує тільки стандартну інформацію.
  • Улюблені аніме: AnimeHub-MPA має можливість додати аніме в улюблене, завдяки localStorage. AnimePotik-SPA такоі можливості не має.
  • Фільтри: AnimeHub-MPA дозволяє вибирати декілька фільтрів одночасно, в залежності від можливостей Jikan API. AnimePotik-SPA дозволяє одночасно вибрати тільки 1 фільтр.
  • Підтримуваність: AnimeHub-MPA набагато легше підтримувати та розширювати ніж AnimePotik-SPA, завдяки сучасним фреймворку React та бібліотек Font Awesome і Marked.
  • Архітектура: AnimeHub-MPA на архітектурі MPA (окремий html для кожної сторінки). AnimePotik-SPA на архітектурі SPA (1 html файл).

🛠️ Технічна Реалізація

Код (Frontend):

  • Чистий код: Проєкт написаний виключно на HTML5, CSS3 та чистому JavaScript (Vanilla JS). Відсутність фреймворків та бібліотек демонструє глибоке розуміння того, що відбувається "під капотом".
  • Модульність: Код розділений на окремі файли, кожен з яких виконує свою конкретну функцію (наприклад, `theme.js` керує перемиканням тем). Це забезпечує легку підтримку та масштабованість проєкту.
  • Оптимізація: Код максимально компактний, ефективний та уникає дублювання (принцип DRY - Don't Repeat Yourself).
  • Динамічні дані: Сайт підключений до реального, популярного публічного API – Jikan. (Детальніше про це в розділі "Jikan API").

Дизайн та Адаптивність:

  • Гнучкий макет: У верстці використовуються виключно адаптивні одиниці (`%`, `vh`, `vw`, `vmin`, `vmax`) та медіа-запити для коректного відображення, включно з альбомним режимом.
  • Мобільний пріоритет: Оскільки проєкт розроблявся на телефоні (Android), його вигляд на мобільних пристроях є ідеальним. У зв'язку з відсутністю ПК, я не можу гарантувати такого ж ідеального відображення на десктопних моніторах без додаткового тестування.
  • Пошук балансу: Під час самостійної розробки без макета було приділено багато уваги балансу між візуальною привабливістю та зручністю (UX), з вивченням дизайнерських рішень на інших ресурсах.
  • Еволюція дизайну: Елементи, такі як градієнти, поле пошуку та банери, неодноразово змінювалися для досягнення найкращого результату.
  • Примітка розробника: Як програміст, я надаю перевагу роботі за готовим макетом, оскільки це забезпечує швидкий результат, що відповідає очікуванням клієнта. Розробка власного ідеального дизайну є тривалим і складним процесом.

Jikan API:

  • Опис: Jikan API — це безкоштовний публічний API для отримання детальної інформації про аніме.
  • Обмеження: Jikan надає лише інформаційні дані, тому функціонал перегляду серій (відео-плеєр) відсутній. З цієї ж причини не реалізовано функції, що потребують бекенду, такі як коментарі, оцінки, відгуки та вхід / реєстрацію.
  • Вивантаження даних: Оскільки API видає по 25 аніме за один запит, ця кількість є стандартною для відображення в усіх відповідних секціях сайту.
  • Різноманіття: База даних Jikan містить інформацію про найрізноманітніші аніме (за роками, студіями, жанрами), включно з контентом для дорослих.

❓ FAQ (Часті Запитання)

Відсутність даних:

  • Неповна інформація: У деяких картках аніме може бути відсутня певна інформація (оцінка, віковий рейтинг, рік). Це не помилка коду, а відсутність даних безпосередньо в API Jikan для цих конкретних аніме. Я вирішив не виводити на екран написи на кшталт "Рік: невідомо", щоб не засмічувати інтерфейс.
  • Мова контенту: На жаль, дані з Jikan доступні лише англійською та японською мовами. Однак, завдяки автоматичному перекладу браузера, який підтримується більшістю сучасних веб-оглядачів, ви зможете переглядати сайт своєю рідною мовою.
